# OWASP Top 10 安全检查清单

## 1. 注入攻击（Injection）

- [ ] SQL 查询是否使用参数化查询或 ORM？
- [ ] 是否存在字符串拼接 SQL 的情况？
- [ ] 用户输入是否经过转义和过滤？

## 2. 身份认证失效（Broken Authentication）

- [ ] 密码是否明文存储？（应使用 bcrypt 等加密）
- [ ] 会话令牌是否使用安全随机数生成？
- [ ] 是否有登录失败次数限制？

## 3. 敏感数据泄露（Sensitive Data Exposure）

- [ ] API 密钥、数据库密码是否硬编码在代码中？
- [ ] 敏感数据是否通过 HTTPS 传输？
- [ ] 日志中是否记录了敏感信息？

## 4. XSS 跨站脚本攻击

- [ ] 用户输入是否在渲染前经过转义？
- [ ] 是否使用 dangerouslySetInnerHTML 等危险 API？
- [ ] CSP（Content Security Policy）头是否设置？

## 5. 安全配置错误

- [ ] 是否关闭了调试模式？
- [ ] 错误页面是否暴露了堆栈信息？
- [ ] 默认账户密码是否已修改？

## 6. 易受攻击的过时组件（Vulnerable and Outdated Components）

- [ ] 是否定期扫描依赖包漏洞（npm audit / dependabot）？
- [ ] 存在漏洞的第三方库是否及时升级？
- [ ] 是否移除未使用、废弃的依赖包？
- [ ] 是否监控第三方组件安全公告？

## 7. 访问控制失效（Broken Access Control）

- [ ] 后台管理接口是否做后端权限校验？
- [ ] 用户能否越权查看/修改他人数据（水平越权）？
- [ ] 普通用户能否访问管理员接口（垂直越权）？
- [ ] 前端隐藏操作按钮后，后端是否仍校验权限？

## 8. 软件与数据完整性失效（Software and Data Integrity Failures）

- [ ] CI/CD 流程是否校验依赖包完整性（lock 文件、哈希校验）？
- [ ] 是否直接引入未校验来源的第三方插件、NPM 包？
- [ ] 自动更新功能是否校验文件签名？
- [ ] JWT 等凭证是否严格校验签名，禁用可修改算法？

## 9. 安全日志与监控失效（Security Logging and Monitoring Failures）

- [ ] 登录、注销、修改密码等敏感操作是否记录日志？
- [ ] 日志包含用户ID、操作时间、客户端IP等溯源信息？
- [ ] 多次登录失败、异常访问是否配置告警？
- [ ] 日志是否持久化存储，满足安全审计溯源要求？

## 10. 服务器端请求伪造 SSRF（Server-Side Request Forgery）

- [ ] 服务端发起外部请求时，拦截内网IP、云元数据地址？
- [ ] 用户可控URL参数使用域名/IP白名单校验？
- [ ] 对外请求限制危险协议（禁止 file://、gopher://）？
- [ ] 服务端HTTP请求是否关闭自动跟随重定向？
